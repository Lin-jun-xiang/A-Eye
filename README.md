# A-Eye 起步警示

手機架在擋風玻璃前，開啟 App 後透過相機即時偵測：
- **前車起步** → 聲音 + 震動 + 閃屏提醒
- **紅燈轉綠燈** → 聲音 + 震動 + 閃屏提醒

適用場景：塞車放空、等紅燈發呆時，避免沒跟上。

---

## 技術架構

### 物件偵測

使用 **TensorFlow.js + COCO-SSD**（`lite_mobilenet_v2`）在瀏覽器端執行物件偵測，能辨識 `car`、`truck`、`bus`、`motorcycle`、`traffic light`。

不是用像素比對猜位置，是真的認出「這是一台車」「這是紅綠燈」。

### 前車追蹤 — 狀態機

系統有 4 個明確狀態：

```
IDLE → LOCKING → TRACKING → DEPARTING → IDLE
```

| 狀態 | 意思 | 轉換條件 |
|---|---|---|
| IDLE | 沒有前車 | 偵測到候選車輛 → LOCKING |
| LOCKING | 發現候選，確認中 | 連續 3 幀同一台 → TRACKING；消失 → IDLE |
| TRACKING | 已鎖定，監控移動 | 前車起步 → 警報 → IDLE；消失 → DEPARTING |
| DEPARTING | 前車消失，確認中 | 連續 5 幀沒回來 → 警報 → IDLE；回來 → TRACKING |

為什麼要這樣設計：
- COCO-SSD 偵測不穩定，某幀可能突然偵測不到 → 用 LOCKING / DEPARTING 做緩衝，避免誤判
- 每個狀態的進入/退出條件明確，不會因為程式邏輯順序出錯

### 鎖定追蹤（IoU 匹配）

選定前車後，**不會每幀重新選**。用 IoU（交集比聯集）比對上一幀和當前幀的框，確認是同一台車。

旁邊車道有大卡車經過也不會跳過去追蹤。

### Bbox EMA 平滑

COCO-SSD 每幀的偵測框會抖動 5~10px。如果用原始值計算面積變化，靜止的車也會被誤判為在動。

用 EMA（指數移動平均）平滑：`平滑值 = 0.6 × 上一幀 + 0.4 × 當前幀`，消除抖動。

### 前車起步判定

比較「3.6 秒前的框」和「現在的框」：
- 面積縮小 > 5%（車在遠離）
- 或 Y 座標上移 > 2.5%（車在遠離）

加上**遲滯門檻**：進入移動狀態的門檻是 5%，但要回到 2% 以下才退出，防止在邊界值來回震盪反覆觸發。

需連續 2 幀都超過門檻才真正觸發警報。

### 自車運動判定 — 路面光流

**核心問題：自己在開車時，前車的框也會變化，不能誤報。**

做法：取畫面底部 75%~90% 的路面區域，建立灰度橫條帶，兩幀之間做 1D 互相關找水平偏移量。

- 路面紋理偏移大 → 自己在動 → 暫停所有警報
- 路面紋理不動 → 自己靜止 → 正常偵測

優點：不依賴物件偵測結果，空曠道路也能判定。

### Ego-Lane 前車篩選

不是隨便選最大的車。用消失點原理：同車道的車會靠近畫面水平中心。

評分公式：`面積 × 高斯中心懲罰`，偏離中心越遠分數衰減越快（σ=0.2）。

### 紅綠燈偵測

1. COCO-SSD 先辨識出 `traffic light` 的位置
2. 在該框內分析像素顏色（亮度 > 0.45、飽和度 > 0.25 的像素）
3. 統計紅色/綠色像素比例判定燈號
4. **只在紅→綠狀態轉換時才警報**，不是看到綠色就叫
5. 需連續 2 幀確認，防閃爍誤判
6. 紅綠燈也有 IoU 鎖定追蹤，不會在多個燈號間跳來跳去

## 省電設計

| 做法 | 效果 |
|---|---|
| 偵測頻率 ~1.7 FPS | 不需要 60fps，每 600ms 分析一次夠用 |
| 模型用 `lite_mobilenet_v2` | COCO-SSD 最輕量版本 |
| 光流分析用 120px 寬的縮圖 | 不用原始解析度 |
| 重用 canvas | 不每幀建立新物件 |
| 自車行駛時只追蹤不分析 | 減少不必要計算 |
| Wake Lock | 螢幕不自動關閉 |

---

## 背景運行

瀏覽器限制：切到背景時相機會被系統暫停。

解法：**Picture-in-Picture（子母畫面）**，按 🖼 按鈕讓相機浮在其他 App 上，可以同時用導航或音樂。

螢幕關閉超過 5 分鐘自動停止偵測。

---

## 使用方式

1. 手機瀏覽器開啟此 App
2. 手機架在擋風玻璃前（像導航一樣）
3. 按「開始偵測」
4. 需要切換其他 App 時按 🖼 進入子母畫面

---

## 技術棧

- 純前端 PWA（HTML + JS），無後端
- TensorFlow.js + COCO-SSD
- Web Audio API（音效）
- Wake Lock API（螢幕常亮）
- Picture-in-Picture API（子母畫面）
- Service Worker（離線快取）
